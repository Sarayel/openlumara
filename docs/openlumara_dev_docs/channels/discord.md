# Discord Channel

The `Discord` channel allows users to interact with OpenLumara via a Discord bot. It provides a highly interactive experience, including support for streaming and group chat awareness.

## Features

- **Live Message Streaming**: Implements a sophisticated streaming mechanism that periodically edits Discord messages to show the AI's response as it is being generated.
- **Group Chat Awareness**: When enabled, the bot injects the name of the user who sent the message into the prompt, allowing the AI to distinguish between different participants in a group chat.
- **Reply Support**: Can be configured to use Discord's native "reply" feature when responding to messages.
- **Command Security**: Allows restricting the use of commands (starting with `/`) to a specific authorized user ID.
- **Tool Call Streaming**: Supports streaming tool call arguments as they are being generated by the AI.

## Configuration Settings

| Setting | Description | Default |
| :--- | :--- | :--- |
| `token` | Your Discord Bot token from the Developer Portal. | `None` |
| `authorised_user_id` | The Discord User ID allowed to use commands. | `None` |
| `target_channel_id` | The specific Discord Channel ID the bot will interact in. | `None` |
| `require_mentions` | Whether the bot must be mentioned to respond. | `True` |
| `use_message_streaming` | Whether to stream messages by periodically editing them. | `False` |
| `edit_interval` | The rate (in seconds) for editing messages in streaming mode. | `1` |
| `show_reasoning` | Whether to show the model's internal reasoning. | `False` |
| `stream_tool_calls` | Whether to stream tool call arguments. | `False` |
| `use_replies` | Whether the bot should use Discord's reply feature. | `False` |
| `enable_group_chat` | Whether to inject user names into the prompt. | `True` |
| `startup_message` | A message sent when the bot starts up. | `None` |
| `shutdown_message` | A message sent when the bot shuts down. | `None` |

## Implementation Details

### Streaming Mechanism
The `_stream_to_discord` method manages the real-time feedback loop:
1. It creates a `StreamState` object to track the current message and content.
2. It starts a `periodic_editor` task that updates the Discord message at a configurable `edit_interval`.
3. It consumes tokens from the AI stream and accumulates them in a `pending_content` buffer.
4. Once the stream is complete, it performs a final edit to ensure the full content is displayed.

### Message Processing
Incoming messages are processed by checking:
- **Mention Status**: If `require_mentions` is true, the bot only responds if mentioned or replied to.
- **Command Detection**: Messages starting with the configured prefix are treated as commands and checked against the `authorised_user_id`.
- **Context Injection**: If `enable_group_chat` is true, the user's display name is prepended to the message content.
- **Reply Formatting**: If the message is a reply, the content of the original message is quoted using Markdown blockquotes.
